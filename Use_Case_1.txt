# **Выбор пиццы и оформление заказа**
***

*Цель:* Пользователь оформляет заказ на сайте с доставкой, чтобы не нужно было тратить время на путь в магазин/кафе/ресторан.  
*Акторы:* Пользователь, система

## **_Основной сценарий:_**
***

 1. Пользователь заходит на сайт.
 2. Система отображает меню выбора пиццы.
 3. Пользователь выбирает пиццу.
 4. Система отображает информацию о пицце и список дополнительных ингредиентов.
 5. Пользователь выбирает дополнительные ингредиенты.
 6. Система отображает стоимость пиццы с учетом выбранных   дополнительных ингредиентов.
 7. Пользователь добавляет пиццу в корзину и начинает оформление      заказа.
 8. Система отображает форму авторизации.
 9. Пользователь заполняет данные (авторизация/регистрация).
10. Система отображает форму заполнения адреса доставки.
11. Пользователь заполняет данные об адресе доставки.
12. Система отображает форму выбора времени доставки.
13. Пользователь выбирает время доставки.
14. Система отображает форму выбора способа оплаты заказа.
15. Пользователь выбирает способ оплаты картой онлайн и оплачивает заказ.
16. Система отображает статус заказа.
17. Пользователь получает свой заказ.

## **_Альтернативный сценарий, исключения:_**
***

### **_Пользователь выбирает адрес доставки_** 

 11. Пользователь выбирает адрес из сохраненных в личном кабинете.  
Далее шаг 12 основного сценария

### **_Пользователь выбирает карту оплаты_** 

15. Пользователь выбирает способ оплаты картой онлайн, выбирает банковскую карту из сохраненных в личном кабинете и оплачивает заказ.  
Далее шаг 16 основного сценария.

### **_Оплата при получении заказа картой_** 

15. Пользователь выбирает способ оплаты картой при получении заказа.  
Далее шаг 16,17 основного сценария.
18. Пользователь оплачивает заказ.

### **_Оплата при получении заказа наличными_** 

15.	Пользователь выбирает способ оплаты наличными деньгами при получении заказа.  
Далее шаг 16,17 основного сценария.
18.	Пользователь оплачивает заказ.

### **_Недостаточно денег на карте для оплаты заказа_**

Система отображает ошибку и открывается главный экран
